00124 // decode a frame of audio sample data if possible from the passed-in data block
00125 //
00126 // return value: number of bytes we used from datablock
00127 // possible cases:
00128 //     0 bytes used, 0 samples output (need more data)
00129 //     N bytes used, 0 samples output (resynching the stream, keep going)
00130 //     N bytes used, M samples output (one frame of data)
00131 // note that after opening a file, you will ALWAYS get one N-bytes,0-sample
00132 // frame, because Vorbis always "discards" the first frame.
00133 //
00134 // Note that on resynch, stb_vorbis will rarely consume all of the buffer,
00135 // instead only datablock_length_in_bytes-3 or less. This is because it wants
00136 // to avoid missing parts of a page header if they cross a datablock boundary,
00137 // without writing state-machiney code to record a partial detection.
00138 //
00139 // The number of channels returned are stored in *channels (which can be
00140 // NULL--it is always the same as the number of channels reported by
00141 // get_info). *output will contain an array of float* buffers, one per
00142 // channel. In other words, (*output)[0][0] contains the first sample from
00143 // the first channel, and (*output)[1][0] contains the first sample from
00144 // the second channel.

00241 // decode the next frame and return the number of samples per channel. the
00242 // data is coerced to the number of channels you request according to the
00243 // channel coercion rules (see below). You must pass in the size of your
00244 // buffer(s) so that stb_vorbis will not overwrite the end of the buffer.
00245 // The maximum buffer size needed can be gotten from get_info(); however,
00246 // the Vorbis I specification implies an absolute maximum of 4096 samples
00247 // per channel. Note that for interleaved data, you pass in the number of
00248 // shorts (the size of your array), but the return value is the number of
00249 // samples per channel, not the total number of samples.
00250 
00251 // Channel coercion rules:
00252 //    Let M be the number of channels requested, and N the number of channels present,
00253 //    and Cn be the nth channel; let stereo L be the sum of all L and center channels,
00254 //    and stereo R be the sum of all R and center channels (channel assignment from the
00255 //    vorbis spec).
00256 //        M    N       output
00257 //        1    k      sum(Ck) for all k
00258 //        2    *      stereo L, stereo R
00259 //        k    l      k > l, the first l channels, then 0s
00260 //        k    l      k <= l, the first k channels
00261 //    Note that this is not _good_ surround etc. mixing at all! It's just so
00262 //    you get something useful.
